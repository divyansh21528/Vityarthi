\# Problem Statement

Users need a secure, easy-to-use system for basic banking transactions such as withdrawals, deposits, and balance inquiries. Manual processes and unsecured methods can lead to errors and unauthorized access. A command-line ATM simulation with PIN verification addresses this need.

\# Scope of the Project

The project implements a command-line application simulating ATM operations with in-memory balance and PIN storage. It supports withdrawing and depositing money, balance checking, and secure access via PIN input. It is intended for educational and demonstration purposes.

\# Target Users

\- Beginners learning programming and security concepts.  

\- Educators demonstrating simple banking applications.  

\- Users looking for a basic ATM transaction simulator.

\# High-Level Features

\- PIN verification before sensitive operations.  

\- Withdrawals with balance check for insufficient funds.  

\- Deposits to increase account balance.  

\- Balance inquiry with secure access.  

\- Menu-driven interface with error handling.  

\- Exit option for safe termination of the application.
